# Сопоставление холдингов с условными ГСЗ

Проект для Excel 2016 (русский интерфейс): автоматическое сопоставление названий холдингов из смарт-таблицы `_HOLD_OD` со справочником групп застройщиков `_base_gsz` по гибким текстовым ключам.

## Задача

В рабочей книге Excel есть:

| Таблица | Назначение |
|---------|------------|
| `_HOLD_OD` | Холдинги и показатели ОД; колонка **«Холдинг»** — текст для поиска |
| `_base_gsz` | Справочник ГСЗ: **«Наименование, регион»** и 12 колонок ключей |
| `_UL_HOLD_OD` | Не используется в текущей версии |

Power Query-запрос `_HOLD_OD_GSZ` для каждой строки `_HOLD_OD` ищет совпадения в «Холдинг» по ключам из `_base_gsz` и возвращает:

- **«условное ГСЗ»** — первое совпавшее «Наименование, регион» или `"-"`;
- **«Отладка_совпадения_ГСЗ»** — все совпадения через `"; "` или `"-"`.

Исходная таблица `_HOLD_OD` не изменяется.

## Логика ключей

### Колонки в `_base_gsz`

| Группа | Колонки | Смысл |
|--------|---------|--------|
| AND full | `key_and_full_1` … `_3` | Обязательные подстроки |
| AND not | `key_and_not_1` … `_3` | Обязательные отдельные слова |
| OR full | `key_or_full_1` … `_3` | Альтернативные подстроки |
| OR not | `key_or_not_1` … `_3` | Альтернативные отдельные слова |

В ячейке — **одно слово**; несколько слов — через колонки `_1` / `_2` / `_3`.

### Модификаторы

| Модификатор | Поведение | Пример |
|-------------|-----------|--------|
| `full` | Подстрока в любом месте | `ПИК` → `ПИКНИК`, `ЗАПИКАТЬ` |
| `not` | Отдельное слово (граница — не буква) | `ПИК` в `ПИКНИК` — **нет** |
| `and` | Все непустые AND-поля; позиции **не пересекаются** | `ПИГ` + `ИГЛА` в `ПИГЛА` — **нет** |
| `or` | Хотя бы одно непустое OR-поле (если OR заполнены) | |

### Формула совпадения строки `_base_gsz`

```
совпадение =
  (все непустые AND-поля выполнены, без пересечения позиций)
  И
  (все OR-поля пусты ИЛИ хотя бы одно OR-поле выполнено)
```

Дополнительно:

- регистр не учитывается (`Text.Lower` + `Text.Trim`);
- буквы: кириллица и латиница;
- строка без ключей не даёт совпадения;
- при нескольких совпадениях: в основной колонке — **первое** по порядку в `_base_gsz`, в отладке — **все**.

## Power Query: подключение `_HOLD_OD_GSZ`

1. Откройте книгу со смарт-таблицами `_HOLD_OD` и `_base_gsz`.
2. **Данные** → **Создать запрос** → **Пустой запрос**.
3. **Расширенный редактор** → вставьте код из `src/PowerQuery/_HOLD_OD_GSZ.pq`.
4. **Готово** → загрузите как подключение или на новый лист.
5. Переименуйте запрос в **`_HOLD_OD_GSZ`**.

### Шаги запроса (основные)

| Шаг | Назначение |
|-----|------------|
| `Источник_HOLD_OD` / `Источник_base_gsz` | Загрузка смарт-таблиц |
| `Справочник_буфер` | Буферизация справочника |
| `fnНормализоватьТекст` | Trim + нижний регистр |
| `fnВсеПозицииFull` / `fnПозицииNot` | Поиск full / not |
| `fnПроверитьAndТокены` | AND без пересечения |
| `fnПроверитьOrБлок` | OR-блок |
| `fnСтрокаGszСовпала` | Проверка одной строки справочника |
| `fnНайтиВсеGsz` | Все совпадения для холдинга |
| `СКолонкой_список_совпадений` | Список совпадений (один проход) |
| `СКолонкой_условное_ГСЗ` / `СКолонкой_отладка` | Итоговые колонки |
| `Без_служебной_колонки` | Удаление `СовпаденияГСЗ` |
| `Результат` | Финальная таблица |

### Совместимость Excel 2016

- Не используются `Table.ExpandRecordColumn` с 4 аргументами и `Table.AddColumn` с `type` — только отдельные шаги `AddColumn`.
- Доступ к колонкам через `Record.Field(_, "имя")`.
- `List.Count = 0` вместо `List.IsEmpty`.

## Генератор ключей `_base_gsz`

Скрипт `src/generate_gsz_keys.py` читает `КейЛОАД.txt` и создаёт Excel локально.

### Запуск

```bash
pip install openpyxl   # один раз
python3 src/generate_gsz_keys.py
```

Результат: `output/_base_gsz_keys.xlsx` (смарт-таблица `_base_gsz`).

### Правила подбора ключей

1. Берётся часть **до запятой** (`ПИК, г.Москва` → `ПИК`).
2. Дефис разбивает на части: `ВКБ-Новостройки` → `ВКБ` + `Новостройки` (не целая строка с дефисом).
3. Убираются служебные слова: ГК, Группа, Холдинг, Фонд и т.д.
4. Короткие токены (≤5 букв) → `not`, длинные → `full`.
5. AND до 3 токенов; проверка уникальности по той же логике, что в PQ.
6. Если уникальность невозможна — одинаковые ключи + `ключи_задублированы = да`.

### Колонки выходного Excel

| Колонка | Описание |
|---------|----------|
| `Наименование, регион` | Исходная строка |
| `key_and_full_*` … `key_or_not_*` | Ключи |
| `ключи_задублированы` | `да` — не удалось развести однозначно |
| `комментарий_ключей` | Причина |

### config.json

| Параметр | По умолчанию | Описание |
|----------|--------------|----------|
| `input_file` | `КейЛОАД.txt` | Входной список |
| `output_file` | `output/_base_gsz_keys.xlsx` | Выходной Excel |
| `min_token_length` | `2` | Минимальная длина токена |
| `short_token_not_max_len` | `5` | Порог для режима `not` |
| `common_token_threshold` | `80` | Зарезервировано |

## Тестовые сценарии

| # | Холдинг | Ключи | Ожидание |
|---|---------|-------|----------|
| T1 | `ГК ПИК` | and_full: `ПИК` | совпадение |
| T2 | `ПИКНИК` | and_not: `ПИК` | нет |
| T3 | `ПИКНИК` | and_full: `ПИК` | совпадение |
| T4 | `ПИГ ИГЛА` | and: `ПИГ`, `ИГЛА` | совпадение |
| T5 | `ПИГЛА` | and: `ПИГ`, `ИГЛА` | нет (пересечение) |
| T6 | `пик строй` | and_full: `ПиК` | совпадение (регистр) |
