# Бизнес-требования: сопоставление холдингов с условными ГСЗ

## 1. Контекст

В Excel-файле (версия 2016, русский интерфейс) ведутся смарт-таблицы с данными по холдингам и справочником групп застройщиков (ГСЗ).  
Задача — автоматически определить, какому условному ГСЗ из справочника соответствует название холдинга, по набору текстовых ключей с гибкой логикой поиска.

**Инструмент реализации:** один запрос Power Query (язык M), все шаги видны отдельно.

---

## 2. Участники данных

### 2.1 Справочник `_base_gsz`

| Поле | Тип | Описание |
|------|-----|----------|
| `Наименование, регион` | текст | Человекочитаемое имя условного ГСЗ; **результат** при совпадении |
| `key_and_full_1` … `3` | текст, необяз. | Обязательные подстроки (режим full) |
| `key_or_full_1` … `3` | текст, необяз. | Альтернативные подстроки (режим full) |
| `key_and_not_1` … `3` | текст, необяз. | Обязательные отдельные слова (режим not) |
| `key_or_not_1` … `3` | текст, необяз. | Альтернативные отдельные слова (режим not) |

### 2.2 Целевая таблица `_HOLD_OD`

| Поле | Описание |
|------|----------|
| `Холдинг` | Текст для поиска |
| `ID холдинга` | Сохраняется |
| `Sum([ОД текущий год])` | Сохраняется |
| `Sum([ОД прошлый год])` | Сохраняется |
| **`условное ГСЗ`** | **Новое поле** — результат сопоставления или `"-"` |

### 2.3 Не в scope v1

Таблица `_UL_HOLD_OD` — не используется в первой версии.

## 3. Правила поиска

### 3.1 Регистронезависимость

Сравнение без учёта регистра.  
`ПиК` в ключе и `пИк` в «Холдинг» — **совпадение**.

### 3.2 Режим `full` — подстрока

Ключ входит в текст холдинга как непрерывная подстрока.

### 3.3 Режим `not` — отдельное слово

Ключ должен быть **отделён от других букв** не-буквенными символами или границами строки.

**Буквы:** кириллица и латиница (предложение по умолчанию).

**Не-буквы (разделители):** пробел, `_`, цифры, `.`, `,`, `-`, `«`, `»`, скобки и т.д.

### 3.4 Множественные слова в одной ячейке ключа

В одной ячейке может быть несколько ключевых слов (разделитель — **уточняется у заказчика**).

| Тип поля | Логика внутри ячейки | Логика между ячейками одного типа |
|----------|----------------------|-------------------------------------|
| `key_and_*` | Все слова найдены, позиции не пересекаются | **И** — все непустые AND-поля должны выполниться |
| `key_or_*` | Достаточно любого слова | **ИЛИ** — достаточно одного непустого OR-поля |

### 3.5 Пересечение (только для AND)

При проверке нескольких слов в AND-режиме подстроки не должны занимать одни и те же символы текста.

**Пример:** слова `ПИГ` и `ИГЛА` в тексте `ПИГЛА` — **не проходят** AND, т.к. вхождения пересекаются.

### 3.6 Комбинация AND и OR

По исходному ТЗ:

1. Сначала проверяются все AND-условия (full и not вместе в блоке AND).
2. Затем, если заданы OR-поля, проверяется блок OR.

**Черновая формула (ожидает подтверждения):**

```
match =
  AND_block_passed
  AND (
    OR_block_is_empty
    OR OR_block_passed
  )
```

Где:

- `AND_block_passed` — все непустые `key_and_full_*` и `key_and_not_*` удовлетворены; если все AND-поля пусты — блок считается пройденным (нет ограничений).
- `OR_block_passed` — хотя бы одно непустое `key_or_full_*` или `key_or_not_*` удовлетворено.

### 3.7 Пустые значения

| Ситуация | Поведение (предложение) |
|----------|-------------------------|
| Пустая ячейка ключа | Игнорируется |
| Все ключи строки `_base_gsz` пусты | Строка не даёт совпадения |
| Пустой «Холдинг» | Результат `"-"` |
| Ни одна строка `_base_gsz` не подошла | Результат `"-"` |

## 4. Результат сопоставления

Для каждой строки `_HOLD_OD`:

1. Перебрать строки `_base_gsz` (порядок — как в таблице Excel).
2. При первом (или всех — уточняется) совпадении записать `Наименование, регион` в `условное ГСЗ`.
3. Иначе — `"-"`.

Одна строка `_base_gsz` может сопоставиться с несколькими холдингами.  
Один холдинг может сопоставиться с несколькими строками `_base_gsz` — **правило разрешения конфликта уточняется**.

## 6. Критерии приёмки (черновик)

| ID | Сценарий | Ожидание |
|----|----------|----------|
| AC-1 | Холдинг содержит ключ full | `условное ГСЗ` = имя из `_base_gsz` |
| AC-2 | Холдинг не содержит ни один ключ | `условное ГСЗ` = `"-"` |
| AC-3 | Разный регистр ключа и холдинга | Совпадение |
| AC-4 | not-ключ внутри слова (`ПИК` в `ПИКНИК`) | Нет совпадения для not |
| AC-5 | full-ключ внутри слова (`ПИК` в `ПИКНИК`) | Совпадение для full |
| AC-6 | AND: два слова без пересечения | Совпадение |
| AC-7 | AND: два слова с пересечением в тексте | Нет совпадения |
| AC-8 | OR: одно из альтернативных полей сработало | Совпадение |
| AC-9 | Заполнены AND и OR: только AND | Нет совпадения (если подтвердится формула п. 3.6) |
| AC-10 | Все шаги запроса видны в редакторе PQ | Да |

## 7. Согласованные решения (диалог)

| Тема | Решение |
|------|---------|
| Слова в ячейке | Одно слово; несколько — через колонки `_1` / `_2` / `_3` |
| AND + OR | AND обязателен **и** хотя бы один OR (если OR-поля заполнены) |
| Вывод | Новый запрос; `_HOLD_OD` не изменяется |
| Несколько совпадений | В «условное ГСЗ» — **первое** по порядку; в отладке — **все** совпавшие «Наименование, регион» |
| not: алфавит | Кириллица **и** латиница — буквы (`PIK` в «ГК PIK» находится) |
| Все AND пусты | AND-блок пройден; достаточно OR |
| Все OR пусты | OR-блок не требуется; достаточно AND |
| Отладочная колонка | Все совпавшие «Наименование, регион» |
| Пересечение AND | **Между всеми AND-полями строки** (`key_and_full_1` + `key_and_not_2` и т.д.); пример: ПИГ + ИГЛА в «ПИГЛА» — нет совпадения |
| Пробелы | Обрезать (`Text.Trim`) у «Холдинг» и ключей |

### ✅ Подтверждено (пересечение AND)

Пересечение проверяется **между всеми AND-полями строки** (как в исходном ТЗ).  
Пример: `key_and_full_1` = `ПИГ`, `key_and_full_2` = `ИГЛА`, холдинг `ПИГЛА` → **нет совпадения**.

## 8. Уточняющие вопросы к заказчику (архив)

Ответы получены в диалоге; см. раздел 7.

### Q1. Разделитель слов внутри ячейки ключа

Как в одной ячейке записано несколько слов?

- a) Пробел  
- b) Точка с запятой `;`  
- c) Запятая `,`  
- d) Каждая ячейка — всегда одно слово (несколько слов только через разные колонки `_1`, `_2`, `_3`)  
- e) Другое: ___

### Q2. Несколько совпадений для одного холдинга

Если две строки `_base_gsz` подходят к одному «Холдинг»:

- a) Берём **первую** по порядку в таблице  
- b) Берём **последнюю**  
- c) **Склеиваем** все `Наименование, регион` через разделитель (какой?)  
- d) Помечаем спецзначением (например `НЕСКОЛЬКО`)  
- e) Другое: ___

### Q3. Логика AND + OR при частично заполненных полях

Пример: заполнены только `key_and_full_1` и `key_or_full_1`, остальные пусты.

Подтвердите формулу:

- **Вариант A:** AND обязателен **и** OR обязателен, если хотя бы одно OR-поле непустое.  
- **Вариант B:** Достаточно только AND, OR — необязательное «расширение».  
- **Вариант C:** Другое (опишите).

### Q4. AND full и AND not вместе

Если заполнены и `key_and_full_*`, и `key_and_not_*` — это один общий AND-блок (должны выполниться **все**)?

Предлагаем: **да**, все непустые AND-поля обоих подтипов обязательны.

### Q5. OR full и OR not вместе

Если сработало любое поле из `key_or_full_*` **или** `key_or_not_*` — OR-блок пройден?

Предлагаем: **да**.

### Q6. Алфавит для границ слова (not)

Считать буквами только кириллицу или кириллицу **и** латиницу?

Пример: ключ `PIK` в холдинге `ГК PIK` — должно ли находиться?

### Q7. Имя и тип итогового запроса

- Загружать результат **на место** `_HOLD_OD` (замена запроса таблицы)?  
- Или создать **новый** запрос / лист (например `_HOLD_OD_GSZ`)?

### Q8. Отладочная колонка

Нужна ли дополнительная колонка, например `ID совпавшего GSZ` или номер строки справочника — для ручной проверки?

### Q9. Пример реальных данных

Можете приложить 3–5 строк `_base_gsz` с заполненными ключами и 3–5 строк `_HOLD_OD` с ожидаемым результатом — ускорит проверку.

## 8. Глоссарий

| Термин | Значение |
|--------|----------|
| ГСЗ | Группа застройщиков |
| Условное ГСЗ | Наименование из справочника, найденное по ключам (не обязательно официальная привязка) |
| full | Поиск подстроки |
| not | Поиск отдельного слова (границы) |
| AND | Все условия блока должны выполниться |
| OR | Достаточно одного условия блока |
